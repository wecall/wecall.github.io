PHP编码规范

By Ron Chen

发表于 2015-01-09

文章目录

规范的代码在团队合作中尤其重要，很多开发人员不愿意看别人的代码，是因为种种管理规范的人员工作没有做好，最后都施加到了程序员身上去解决面对这些窘境。

目前毕竟浏览的软件项目质量管理的软件【sonar】,里面规范了各种语言代码应该遵循的规范，个人感觉身为团队的一员，有必要在这方面提升一下自己的编程素质。

  • 类的大括号在后面不是另起一行; eg： class user{
  • 变量名首字母小写,采用驼峰模式(小驼峰法); eg: userAge [a-z][a-zA-Z0-9]*
  • 类文件名只允许字母,首字母大写(大驼峰法);
  • 注释要另起一行,不要跟在代码后面;
  • 移除已经注释的代码片段;
  • switch至少包含3个case,否则使用if;
  • switch要加 default;
  • if等嵌套不能超过3次;
  • 类中的方法不能超过20个,否则考虑拆分;
  • 移除没用的参数;
  • 移除没用的变量;
  • if必须要跟else;
  • if总是跟着大括号;
  • 代码中不要有太多的return;
  • 每行结尾不允许有多余的空格;
  • 每行代码长度控制在80个字符以内,最长不要超过120个字符;
  • 使用4个空格作为缩进，而不使用tab缩进,编辑器最好事先设定;
  • 函数中用逗号来分隔参数，所有的参数与前面的逗号之间要空格(第一个参数除外);
  • 除了参数之间要使用空格外，所有操作符之间都要使用空格，包括字符连接符(.);
